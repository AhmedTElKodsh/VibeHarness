# Workflows

Workflow documentation for VibeHarness.

The implemented starter workflow is `default-feature`, generated into `.vibeharness/workflows/default-feature.yaml` by `bun src/cli.ts init`. The mock policy fixture also uses `policy-blocked`.

The engine currently loads workflows from the target project's `.vibeharness/workflows/` directory, not from this repository-level `workflows/` directory.
